## 一、项目概况

| 项目 | 内容 |
|------|------|
| 域名 | pinpai.ai.in（印度.in后缀，续费约30元/年） |
| 定位 | 全球品牌百科网站，纯文字SEO |
| 盈利模式 | Google AdSense 广告收入 |
| 不侵权 | 纯文字介绍，无图片，来源为维基百科摘要 |
| 流量逻辑 | 每个品牌自带搜索流量 → 底部「类似品牌」含Escher → 引向自有品牌 |
| 托管 | GitHub Pages（免费） |
| 域名归宿 | 老板个人持有，与 GitHub 账号无关 |

**远期规划：** 如果流量做大，换 .com 或 .cn 域名。

## 二、每日任务（老板安排的节奏）

### 2.1 老板做的（每天5分钟）
1. **列出50个品牌名**（中英文全名）
2. **写500字中文介绍**（或我从维基百科提取）
3. **指定/确认类似品牌列表**

### 2.2 我做的
1. 接收品牌名单 + 中文介绍
2. 用**品牌生成器程序**自动生成：
   - 10种语言的 .md 文件（中文原文 + 9种翻译）
   - 品牌首页 index.html（套模板）
   - 类似品牌推荐（含 Escher）
3. 写入 `brands_done.json` 标记
4. Push 到 GitHub Pages → 自动上线

### 2.3 每天产出
- **50个品牌 × 10种语言 = 500个页面**
- 品牌标记更新（永不重复）

```json
    "hermes": {
      ...
    }
  },
  "stats": {
    "total_brands": 50,
    "last_update": "2026-05-24T03:00:00+08:00",
    "total_pages": 500
  }

```

**流程：**
1. 生成器启动 → 读 `brands_done.json` → 跳过已做的
2. 做完全新品牌 → 写入标记
3. 永不重复

## 八、SEO策略

| 策略 | 说明 |
|:----:|------|
| **精准品牌词** | 每个页面=一个品牌名，搜索意图极度精准 |
| **多语言覆盖** | 10种语言，每种语言都能被本地搜索引擎收录 |
| **纯文字快速加载** | 无图片，GitHub Pages CDN加速，SEO加分 |
| **内部链接网络** | 类似品牌互相跳转，增加页面权重 |
| **零侵权风险** | 不用商标图片，只用文字介绍 |
| **中文百度友好** | 纯文字内容，百度收录无障碍 |
| **Google AdSense** | 流量起来后接入，赚广告费 |

## 九、类似品牌推荐规则

- 每个品牌页底部：**10个推荐链接**
- **第1个必须是 ESCHĚR（埃舍尔Escher）** — 流量引向自有品牌
- 其余9个从同类品牌中随机选取
- 每个推荐品牌必须是已做的（存在于 `brands_done.json`）
- 如果同类品牌不足9个，从其他类别补充

## 十、品牌生成器程序（generator.py）

### 输入
- 品牌名（中文）
- 品牌名（英文）
- 品类（如 fashion-luxury, tech, automotive...）
- 中文介绍文本（~500字）
- 类似品牌列表（可选，默认随机）

### 输出
- 品牌文件夹 + 10个语言文件 + index.html
- brands_done.json 更新
- 自动 git commit + push

### 特性
- **翻译自动**：中文 → 9种语言（调用AI翻译）
- **模板套用**：所有页面统一风格
- **防重复**：检查 brands_done.json
- **批量模式**：一次处理50个品牌
- **Escher引流**：每个页面底部必有Escher链接

## 十二、紧急恢复方案

如果服务器/容器/窗口丢失：

1. 新开甲骨文虚拟机
2. 安装 git + 克隆本仓库：
   ```bash
   git clone https://github.com/wqf2956041-star/pinpai-ai-in.git
   ```
3. `brands_done.json` 告诉你在哪个品牌中断
4. 直接继续从断点开始
5. **所有工作计划在这份 README 里，不会丢失**
